Introduction to ARM

ARM Introduction

ARM is a RISC (Reduced Instruction Set Computing) architecture.
It operates with registers with a few instructions for loading and saving data from/to memory.

ARM in comparison to other architectures such as x86, is simpler in general terms.
It also has other advantages such as better power consumption, more registers, and easier manufacture.

All those advantages allow phone companies or similar use cases be interested in this architecture.
Both OS X and Android run ARM.

Example code


        .data
        dato1:  .word 2	@ First value
        dato2:  .word 5	@ Second Value
        sum:    .word 0 @ Result
        
        .text
        .global main	        @ Set main as global so it can be called from outside
        main:	ldr r0, =dato1	@ Load the direction of the first value in r0
                ldr r1, [r0]	@ Load the first value in r1 (direction found in r0)
                ldr r0, =dato2	@ Load the direction of the 2nd value in r0
                ldr r2, [r0]	@ Load the second value in r2 (direction found in r0)
                add r0, r1, r2	@ Sum both registers and save in r0
                ldr r3, =sum	@ Load the direction of value in r3
                str r0, [r3]	@ Store the result (r0) to sum (direction found in r3)
        fin:	bx lr           @ End the program, go back to the code that called "main"

If you were able to follow with the comments you would see that all we are doing is calculating a sum in ARM.

The code isn't actually very hard, but you must visualize some concepts first such as registers and have very clear that the VALUE is not the DIRECTION of a variable.

You will be able to differentiate different parts in the code too: the .data, and the .text

All you find in the .data is... well... basically data! They are the most similar concept to variables that we have in ARM.

Note: there are no variables as you know them from other languages, you have to work with registers[1]. Here they act like references in memmory.

On the other hand, there is the .text part. There you will basically find the "logic" of the program. There are a lot of instructions but you always have the good old documentation to read what each one does. Here we don't do anything crazy, just:

  • ldr: LoaD Register --> Loads from a register
  • add: ADDs two registers
  • str: SToRe --> Stores a value in a register
  • bx: Branch and eXchange --> Similar to a return in other languages

Well I think that would be enough for a quick introduction. If you want to know more about it you can always check my course on YouTube where I teach you a lot of interesting advices and concepts starting from the basics.
